## 判定区分の定義

- 本運用: 通常運用で継続可
- 条件付き運用: 実行は可能だが、継続条件の監視を前提に運用可
- 保留: 実行停止中。再評価完了まで本運用・条件付き運用の対象外
- 用語統一: この文書の「本運用」は、引き継ぎ系 docs で使う「定着運用」と同義で扱う

## 最終運用開始フェーズ 本運用開始対象（確定）

### 本運用開始する機能

- 一覧表示
  - 理由: 参照系で副作用がなく、継続稼働が安定
- 詳細表示
  - 理由: 参照系で副作用がなく、継続稼働が安定
- Dashboard
  - 理由: 参照系で副作用がなく、更新反映も安定
- Audit Log 参照
  - 理由: 参照系で副作用がなく、表示整合が安定
- Settings 参照
  - 理由: 参照系で副作用がなく、表示整合が安定
- Connect
  - 理由: 実行結果と状態反映が安定
- Disconnect
  - 理由: 実行結果と状態反映が安定

### 条件付き運用で開始する機能

- Settings 保存
  - 理由: 保存後の値一致・再表示一致の継続監視が必要

- Freeze
  - 理由: 状態変更系のため、confirmed・Audit Log の継続監視が必要
- Unfreeze
  - 理由: 状態復帰系のため、status 整合の継続監視が必要
- Export
  - 理由: 最終危険操作段階のため、LockGuard・認証・fail-safe の毎回確認が必要
- Delete
  - 理由: 不可逆操作のため、Export 条件に加えて削除後整合の毎回確認が必要

- 補足: Freeze / Unfreeze / Export / Delete は、開始時点では「条件付き運用」であり「保留」ではない

### 今回も保留する機能

- （現時点で保留対象なし）
- 保留戻し条件: 上記機能で重大異常または再発異常が出た場合は当該機能を保留に戻す

## 最終運用初回監視フェーズ 異常兆候・初動停止条件（固定）

- 一覧/詳細表示崩れ
  - 目安: 継続監視（単発軽微） / 機能保留へ戻す（再発） / 全面停止（操作不能）
- Dashboard 不整合
  - 目安: 継続監視（単発遅延） / 機能保留へ戻す（再発） / 全面停止（全体波及）
- Audit Log が増えない
  - 目安: 機能保留へ戻す（1件欠落） / 全面停止（連続欠落）
- Settings 保存不整合
  - 目安: 機能保留へ戻す（巻き戻り・空欄化） / 全面停止（保存系波及）
- Connect / Disconnect 不整合
  - 目安: 機能保留へ戻す（状態不一致） / 全面停止（認証・状態系波及）
- Freeze / Unfreeze 不整合
  - 目安: 機能保留へ戻す（status 不整合） / 全面停止（confirmed すり抜け・Audit Log 欠落）
- Export / Delete 不整合
  - 目安: 機能保留へ戻す（結果不一致・3画面不整合） / 全面停止（不可逆影響・認証すり抜け）
- LockGuard / 認証 / fail-safe 不備
  - 目安: 全面停止（confirmed 未チェック・認証不足で実行通過）
- message 不整合
  - 目安: 継続監視（単発軽微） / 機能保留へ戻す（再発） / 全面停止（逆転表示連続）
- 再描画崩れ
  - 目安: 継続監視（単発軽微） / 機能保留へ戻す（空白化・再発） / 全面停止（全画面波及）
- 一覧/詳細/管理パネルの状態食い違い
  - 目安: 機能保留へ戻す（対象機能限定） / 全面停止（複数機能波及）

## 本運用継続判定フェーズ 継続・条件付き継続・保留戻し候補（固定）

### 継続してよい機能
- 一覧表示 / 詳細表示 / Dashboard / Audit Log 参照 / Settings 参照（参照系として安定）
- Connect / Disconnect（低リスク更新系として結果整合が安定）

### 条件付き継続にする機能
- Settings 保存（保存後値整合・再表示整合の継続監視が必要）
- Freeze / Unfreeze（状態変更系のため confirmed・Audit Log・状態整合の継続監視が必要）
- Export（危険操作系のため LockGuard / 認証 / fail-safe の毎回確認が必要）
- Delete（不可逆操作のため Export 条件 + 3画面整合の毎回確認が必要）

### 保留へ戻す候補
- Freeze / Unfreeze（confirmed すり抜け・status 不整合・Audit Log 欠落で保留戻し候補）
- Export / Delete（認証不足通過・LockGuard 誤作動・結果不一致・3画面不整合で保留戻し候補）

## 機能別完成状態参照欄

- 参照元: `docs/gui_handoff_feature_status.md` の「最終完成報告フェーズ 機能別完成状態一覧」
- 完成状態の区分:
  - 完了: 一覧表示 / 詳細表示 / Dashboard / Audit Log 参照 / Settings 参照 / Connect / Disconnect
  - 条件付き完了: Settings 保存 / Freeze / Unfreeze / Export / Delete
  - 保留: 現時点では該当なし
- 運用状態との対応:
  - 完了 = 本運用
  - 条件付き完了 = 条件付き運用
  - 保留 = 保留（実行停止）
- 更新時ルール:
  - 機能の区分変更時は本欄と `docs/gui_handoff_feature_status.md` を同時更新する
  - 保留が発生した場合は `docs/gui_known_gaps.md` も同時更新する
