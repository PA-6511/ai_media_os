## 0. 設計方針サマリ

| 原則 | 内容 |
|------|------|
| 1画面1責務 | 各画面は単一の目的のみを持つ。横断機能は専用画面に切り出す |
| Dashboard 禁止操作 | Dashboard は要約＋ナビゲーションのみ。状態変更操作は一切置かない |
| 危険操作の完全分離 | Export / Delete は専用画面 `/danger-zone` に隔離。通常画面にボタンを置かない |
| 接続操作の明示的ハンドシェイク | 接続/切断はすべて相互確認モーダルを経由する。即時実行ボタンは存在しない |
| ログ閲覧と設定変更の分離 | Audit Log（閲覧専用）と Settings（変更専用）は別画面 |
| BlockAI 増加耐性 | 一覧はフィルタ・検索・仮想スクロール対応。カード数上限なし |
| アダルト系フラグの隔離 | コンテンツポリシー設定（実験AI専用フラグ）は Settings 内に限定 |

## 2. 各画面の目的（1文）

| 画面 | 目的 |
|------|------|
| **S1 Dashboard** | システム全体のヘルス・接続状況・アラートを一覧表示する（操作なし） |
| **S2 BlockAI管理** | BlockAIの一覧・個別設定・コンテンツポリシーを管理する |
| **S3 接続/切断管理** | 合議AI・BlockAIの接続/切断を相互確認ハンドシェイクを経て実行する |
| **S4 危険操作管理** | Export / Delete をパスワード再入力＋2FAの二重認証で実行する |
| **S5 監査ログ** | 全操作の記録を閲覧・検索する（読取専用、操作不可） |
| **S6 セキュリティ設定** | 2FA・アクセス制御・セッション設定・実験AIポリシーを管理する |

## 3. 画面ごとの主要UI要素

---

### S1: Dashboard `/dashboard`

**目的:** 要約表示のみ。ボタン＝「詳細へ」ナビゲーションのみ（状態変更ゼロ）

| UI要素 | 説明 |
|--------|------|
| 接続状況サマリカード | BlockAI 接続中 N件 / 切断中 N件 / エラー N件 |
| 合議AIステータスインジケーター | 接続中🟢 / 切断中🔴 / ハンドシェイク中🟡 |
| システムヘルスバー | CPU / メモリ / API平均応答時間（直近5分） |
| アラートバナー | WARNING / ERROR がある場合のみ表示。クリックで監査ログへ |
| 直近操作サマリ | 最新5件の操作（タイムスタンプ＋操作種別のみ表示）。クリックで監査ログへ |
| 「Block管理へ」ボタン | `/blocks` へのナビゲーション |
| 「接続管理へ」ボタン | `/connections` へのナビゲーション |

> ⚠️ **禁止事項:** Dashboard に接続ボタン・削除ボタン・設定変更欄を置かない

---

### S2: BlockAI管理 `/blocks`

**目的:** BlockAIの全量管理。増加耐性のある一覧設計

| UI要素 | 説明 |
|--------|------|
| 検索バー | BlockAI名・種別でインクリメンタル検索 |
| フィルタチップ | 接続状態（接続中/切断中/エラー）/ コンテンツポリシー（標準/実験） |
| ソートヘッダー | 名前・種別・接続状態・最終ハンドシェイク日時でソート可能 |
| BlockAI一覧テーブル/グリッド | 仮想スクロール対応（100件以上でも遅延なし） |
| 各行の表示項目 | BlockAI名 / 種別 / 接続状態バッジ / コンテンツポリシーバッジ / 最終ハンドシェイク |
| 「接続管理へ」行アクション | 該当BlockAIを選択した状態で `/connections` へ遷移（直接実行しない） |
| 詳細サイドシート | 行クリックで右スライドパネル表示。設定値・スペック・備考を表示 |
| ページネーション or 仮想スクロール | BlockAI 50件以上で自動切り替え |

> ⚠️ **禁止事項:** この画面に接続/切断の実行ボタン・Export/DeleteボタンをUIとして配置しない

---

### S3: 接続/切断管理 `/connections`

**目的:** ハンドシェイク付き接続/切断の唯一の実行画面

| UI要素 | 説明 |
|--------|------|
| 操作対象セレクター | 合議AI / 特定BlockAI（名前で検索）/ 全BlockAI（バルク） |
| 操作種別ラジオ | 「接続する」 / 「切断する」 |
| 「実行する」ボタン | クリックでハンドシェイク確認モーダル表示（即時実行しない） |
| **ハンドシェイク確認モーダル** | 操作内容サマリ表示 |
| └ 送信側確認チェック | 「送信側: 接続命令を送信しました」チェックボックス |
| └ 受信側確認チェック | 「受信側: 接続命令を受信しました」チェックボックス |
| └ 「実行を確定する」ボタン | 両チェックがONの場合のみ活性化 |
| ハンドシェイク進捗インジケーター | PENDING → HANDSHAKE → DONE / FAILED のステップ表示 |
| 結果トースト通知 | 成功 / 失敗をインラインで表示 |
| 直近操作履歴パネル | このセッションの接続/切断履歴（最新10件）。監査ログへのリンク付き |

> ⚠️ **禁止事項:** ハンドシェイクをバイパスして即時実行するパスを作らない

---

### S4: 危険操作管理 `/danger-zone`

**目的:** Export / Delete の完全分離・二重認証付き実行

| UI要素 | 説明 |
|--------|------|
| DANGER ZONEヘッダー | 赤背景・警告アイコン付き。「この操作は取り消せません」明示 |
| **Exportセクション** | |
| └ エクスポート対象選択 | BlockAI名・日付範囲・データ種別 |
| └ 「Exportを開始」ボタン | → パスワード入力モーダル → 2FAコード入力モーダル → 最終確認 |
| **Deleteセクション** | |
| └ 削除対象選択 | BlockAI名入力（テキストフィールドに対象名を再入力必須） |
| └ 「Deleteを開始」ボタン | → パスワード入力モーダル → 2FAコード入力モーダル → 対象名再入力 → 最終確認 |
| 認証フロー（共通） | パスワード入力 → 2FAコード入力 → 対象確認 → 「実行する（取消不可）」ボタン |
| 操作完了ステータス | 結果（成功/失敗）＋監査ログへのリンクを表示 |

> ⚠️ **アクセス制御:** このページはAdminロール限定。ロール不足時は403表示  
> ⚠️ **禁止事項:** このページへのリンクをナビゲーションバーの目立つ場所に置かない（設定メニュー内のみ）

---

### S5: 監査ログ `/audit-log`

**目的:** 全操作履歴の閲覧専用。操作ボタン一切なし

| UI要素 | 説明 |
|--------|------|
| ログ一覧テーブル | タイムスタンプ / 操作者 / 操作種別 / 対象 / 結果 / IPアドレス |
| 日付範囲フィルタ | 開始日〜終了日のDatePicker |
| 操作種別フィルタ | 接続 / 切断 / Export / Delete / 2FA / 設定変更 / ログイン / ログアウト |
| 結果フィルタ | 成功 / 失敗 / 認証エラー |
| フリーテキスト検索 | 操作者名・対象名・エラーメッセージで検索 |
| ログ詳細パネル | 行クリックで詳細展開（フルJSONまたは整形表示） |
| ページネーション | 50件/ページ（固定） |

> ⚠️ **禁止事項:** この画面に操作ボタン・設定変更UIを一切置かない  
> ⚠️ **Exportは危険操作管理へ誘導:** 「ログをエクスポートする場合は危険操作管理ページへ」リンクのみ表示

---

### S6: セキュリティ設定 `/settings/security`

**目的:** 認証設定・アクセス制御・実験AIポリシーの管理（変更は保存確認必須）

| UI要素 | 説明 |
|--------|------|
| 2FAセクション | 有効化 / 無効化 / バックアップコード再生成 |
| アクセス制御セクション | 許可IPアドレス一覧（CIDR対応）/ 追加・削除 |
| セッション設定セクション | タイムアウト時間（分）/ 同時ログイン上限数 |
| **実験AIポリシーセクション** | アダルト系コンテンツ扱いの設定（実験AIフラグのみで有効化可） |
| └ 実験AI有効化トグル | デフォルト: OFF。ONにする際は確認モーダル表示 |
| └ 対象AI選択 | 実験AIとして登録されたBlockAIのみ選択可能 |
| 「変更を保存」ボタン | パスワード再入力モーダルを経て保存（設定変更もパスワード必須） |

> ⚠️ **アダルト系コンテンツ:** 実験AI以外での設定が物理的に不可能なUIにする（フラグ未設定のBlockAIは選択肢に表示しない）

## 4. 画面遷移の概要

```
[ログイン画面]
      │
      ▼
[S1: Dashboard] ──────────────────────────────────────────┐
      │                                                     │
      ├──「Block管理へ」──────────────────── [S2: BlockAI管理]
      │                                           │
      │                                    「接続管理へ」（行アクション）
      │                                           │
      ├──「接続管理へ」──────────────────── [S3: 接続/切断管理]
      │                                           │
      │                                    ハンドシェイクモーダル
      │                                    (画面遷移なし・モーダル内完結)
      │
      ├──「監査ログ」──────────────────────[S5: 監査ログ] (READ ONLY)
      │
      └──[設定メニュー（管理者のみ表示）]
              │
              ├── [S6: セキュリティ設定]
              │
              └── [S4: 危険操作管理] ← 目立たない場所に配置
```

### 遷移ルール

| ルール | 内容 |
|--------|------|
| 接続/切断は S3 経由のみ | S2 の行アクションは S3 へのリンク。S3 以外で実行できない |
| S4 へのリンクは設定メニュー内のみ | ヘッダーナビゲーションには表示しない |
| S5 は全画面のアラートバナーからアクセス可 | 監査ログは常にアクセス可能（閲覧ロール以上） |
| 設定系（S6/S4）はAdminロール必須 | ロール不足時は403ページにリダイレクト |
| ログアウト後は必ずログイン画面に戻る | ブラウザバックでセッション再利用不可 |

## 6. 確認ポイントチェックリスト

### ✅ 画面が増えすぎていないか
- MVP 6画面（後回し2画面）。最小構成を維持。

### ✅ 1画面1責務になっているか
- Dashboard: 要約のみ
- S2: BlockAI管理のみ
- S3: 接続/切断実行のみ
- S4: 危険操作のみ
- S5: ログ閲覧のみ
- S6: セキュリティ設定のみ

### ✅ 危険操作が通常操作と混ざっていないか
- S4 は完全独立画面
- S4 へのリンクは設定メニュー内のみ（ヘッダーナビゲーションには非表示）
- S2・S3 に Export/Delete ボタンを置かない

### ✅ Dashboard に何でも詰め込みすぎていないか
- Dashboard に操作ボタンなし
- 表示項目: 接続状況・ヘルス・アラート・直近5件の操作履歴のみ
- 「詳細はこちら」系のナビゲーションリンクのみ提供

### ✅ 将来の BlockAI 増加時に一覧性が保てるか
- S2: 仮想スクロール・フィルタ・検索・ソートを初期実装
- S3: 操作対象の「全BlockAI」バルク操作を設計に含める

### ✅ ログ閲覧と設定変更が分離されているか
- S5（監査ログ）: READ ONLY、設定変更UI一切なし
- S6（設定）: 変更のみ、ログ表示なし
