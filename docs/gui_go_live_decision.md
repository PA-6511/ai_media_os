# GUI 稼働開始判定結果

- 判定日: 2026-03-23

## 段階解放レベル分け（稼働拡大フェーズ）

### Level1: 参照系

- 一覧表示: 読み取り専用で影響が限定的
- 詳細表示: 読み取り専用で状態変更なし
- Dashboard: 集計参照のみで可逆性不要
- Audit Log 参照: 参照のみで監査確認に必要
- Settings 参照: 値確認のみで副作用なし

## Level1 参照系 安定監視ポイント（Dashboard / Audit Log / Settings）

- Dashboard 数値が表示され続けるか
  - 正常: 集計値が表示され続け、再読込後も消えない
  - 異常兆候: Dashboard 数値が空欄化する、描画されない
- Audit Log 一覧が崩れず読めるか
  - 正常: 一覧の行・列・順序が維持され、内容を読める
  - 異常兆候: 行欠落、列崩れ、順序崩れで追跡不能になる
- Settings 参照値が空にならないか
  - 正常: 取得した参照値が安定表示される
  - 異常兆候: 値が空欄化する、表示が初期値へ戻る
- 参照系表示が古いまま固まらないか
  - 正常: 再取得後に一覧・詳細・Dashboard・Audit Log・Settings が更新される
  - 異常兆候: いずれかが古い表示のまま固定される
- プレースホルダー復帰が適切か
  - 正常: 取得失敗時にプレースホルダーへ戻り、再取得成功で復帰する
  - 異常兆候: 失敗後に復帰せず空白表示が継続する

### Connect / Disconnect 安定監視ポイント

- Connect 後に connectionStatus が整合するか
  - 正常: 接続実行後、一覧・詳細・管理パネルで状態が「connected」に統一される
  - 異常兆候: 一部の画面で接続状態が反映されず、状態が不一致のまま継続する
- Disconnect 後に connectionStatus が整合するか
  - 正常: 切断実行後、一覧・詳細・管理パネルで状態が「disconnected」に統一される
  - 異常兆候: 一部の画面で切断状態が反映されず、接続状態のまま表示が固定される
- 詳細/一覧/管理パネルで表示が揃うか
  - 正常: Connect/Disconnect の操作直後に、3画面の表示が同一の状態を示す
  - 異常兆候: 画面遷移時に状態表示がズレたり、複数の状態が混在して表示される
- message が適切か
  - 正常: Connect/Disconnect の成功/失敗時に、結果に対応した message が即座に表示される
  - 異常兆候: message が出ない、失敗時も成功 message が表示される
- Audit Log が更新されるか
  - 正常: Connect/Disconnect の操作後、Audit Log に action/target/result が追記される
  - 異常兆候: 操作後も Audit Log に記録が追加されない、または記録内容が誤る
- 参照系が巻き込まれて崩れないか
  - 正常: Connect/Disconnect 実行後も、一覧・詳細・Dashboard・Audit Log が崩れず表示される
  - 異常兆候: 状態操作後に参照系の表示が欠落、崩れ、初期化されるなどの影響が出る

### Level2: 低リスク更新系

- Connect: 可逆操作で復旧しやすい
- Disconnect: 可逆操作で復旧しやすい

### Level3: 条件付き更新系

- Settings 保存: 永続値更新のため整合監視が必要
- Freeze: 業務影響があり fail-safe 前提
- Unfreeze: 業務影響があり fail-safe 前提

#### Level3 条件付き継続 追加監視条件

### Freeze（条件付き継続の場合）

- 継続条件: confirmed 導線を維持したまま freezeStatus が一覧/詳細/管理パネルで一致して維持される
- 追加で見るべき指標:
  - confirmed 未チェック実行件数: 0
  - freezeStatus 不整合件数: 0
  - Audit Log 欠落件数: 0
  - message 一致率: 100%
- 監視頻度: 毎回 Freeze 実行後に即時確認し、監視サイクルごとに再確認
- 継続中止条件: confirmed すり抜け、状態未反映、状態不整合、再描画崩れのいずれかが再発

### Unfreeze（条件付き継続の場合）

- 継続条件: confirmed 導線を維持したまま freezeStatus が active へ復帰し一覧/詳細/管理パネルで一致する
- 追加で見るべき指標:
  - confirmed 未チェック実行件数: 0
  - freezeStatus 復帰失敗件数: 0
  - Audit Log 欠落件数: 0
  - message 一致率: 100%
- 監視頻度: 毎回 Unfreeze 実行後に即時確認し、監視サイクルごとに再確認
- 継続中止条件: confirmed すり抜け、状態未復帰、状態不整合、再描画崩れのいずれかが再発

### Level1 / Level2 巻き込み確認（条件付き継続の場合）

- 継続条件: 一覧表示・詳細表示・Dashboard・Audit Log 参照・Settings 参照/保存・Connect/Disconnect が Freeze/Unfreeze 後も安定表示を維持
- 追加で見るべき指標:
  - 一覧/詳細/Dashboard の崩れ件数: 0
  - Audit Log 参照崩れ件数: 0
  - Settings 参照/保存不整合件数: 0
  - Connect/Disconnect 表示不整合件数: 0
- 監視頻度: Freeze/Unfreeze 各実行直後に毎回確認
- 継続中止条件: 巻き込み崩れや状態不整合が再発し、単発復旧で収束しない

#### Freeze / Unfreeze 警告表示・確認導線 安定監視ポイント

- 警告表示が正しく開くか
  - 正常: Freeze / Unfreeze 操作時に対象操作に対応した警告表示が開く
  - 異常兆候: 警告表示が開かない、別操作の文言で開く、二重に開く
- キャンセルで正しく閉じるか
  - 正常: キャンセル操作で警告表示が閉じ、freezeStatus や他表示に変化が出ない
  - 異常兆候: 閉じない、閉じても背景操作ができない、状態だけ変わる
- 確認チェック未実施時に実行されないか
  - 正常: confirmed 未チェックでは Freeze / Unfreeze が実行されない
  - 異常兆候: 未チェックでも実行できる、または実行要求が通る
- 実行後に警告表示が残らないか
  - 正常: 実行完了後に警告表示が閉じ、次画面操作へ戻れる
  - 異常兆候: 成功/失敗後も警告表示が残る、再操作不能になる
- message が適切か
  - 正常: 警告表示経由の成功/失敗に応じて message が結果どおり表示される
  - 異常兆候: message が出ない、内容が逆転する、同一結果で重複表示される
- 他画面に誤影響が出ないか
  - 正常: 一覧・詳細・Dashboard・Audit Log・Settings・Connect/Disconnect 表示が崩れない
  - 異常兆候: 警告表示の開閉や実行後に他画面が空白化、崩れ、操作不能になる
- LockGuard と混同されないか
  - 正常: Freeze / Unfreeze の警告導線として扱われ、Export / Delete 用 LockGuard 導線とは別挙動である
  - 異常兆候: Export / Delete 用の認証導線や文言が混在して表示される

#### Level3 解放実行フェーズ 停止/巻き戻し条件（厳しめ）

- 確認チェック無しで Freeze / Unfreeze が動く
  - 目安: 即停止
  - 判定: confirmed 未チェックで1回でも実行された時点で停止し、Level2 へ巻き戻し判断
- Freeze / Unfreeze 後に状態が変わらない
  - 目安: 即停止
  - 判定: 実行完了後も freezeStatus が遷移せず、再操作でも改善しない
- freezeStatus が不整合
  - 目安: 即停止
  - 判定: 一覧・詳細・管理パネルで同一対象の freezeStatus が一致しない
- Audit Log が更新されない
  - 目安: 条件付き停止
  - 判定: 単発欠落は継続監視、再発または連続欠落で停止し巻き戻し判断
- message が不整合
  - 目安: 条件付き停止
  - 判定: 単発不整合は継続監視、同一不整合の再発で停止し巻き戻し判断
- 再描画が崩れる
  - 目安: 即停止
  - 判定: 一覧・詳細・Dashboard・Audit Log の崩れが再操作後も復帰しない
- 一覧/詳細/管理パネルの状態が食い違う
  - 目安: 即停止
  - 判定: 同一対象の状態が3画面で一致せず収束しない
- 警告表示が閉じない / 誤作動する
  - 目安: 条件付き停止
  - 判定: 単発は継続監視、再発または操作中断不能を伴う場合は停止し巻き戻し判断

### Level4: 危険操作系

- Export: 取り消し不能要素があり LockGuard 前提
- Delete: 取り消し不能で誤操作影響が大きい

## 稼働拡大後の停止条件（厳守）

- 再描画崩れ
  - 目安: 即停止
  - 判断基準: 一覧・詳細・Dashboard・Audit Log のいずれかで表示欠落/崩壊が発生し、再操作でも復帰しない
- message 不整合
  - 目安: 条件付き停止
  - 判断基準: API 結果と異なる成功/失敗 message が発生。単発は継続監視、同種事象が再発したら停止
- 状態更新不整合
  - 目安: 即停止
  - 判断基準: connectionStatus / freezeStatus / 詳細表示の状態が相互に不一致となり、操作完了後も解消しない
- Audit Log 更新失敗の継続発生
  - 目安: 条件付き停止
  - 判断基準: 単発欠落は継続監視、連続欠落または同一操作で再発した場合は停止
- Settings 保存不整合
  - 目安: 即停止
  - 判断基準: 保存成功表示にもかかわらず再読込で値が戻る、または失敗時に state が書き変わる
- 危険操作 fail-safe 不備
  - 目安: 即停止
  - 判断基準: confirmed 未チェックや認証不足でも危険操作 API が実行される
- LockGuard 誤作動
  - 目安: 条件付き停止
  - 判断基準: 単発誤作動は即時切り戻しの上で継続監視。再発または中断不能事象を伴う場合は停止

## Level別 巻き戻し条件・停止条件

### Level1: 参照系

- 再描画崩れ: 一段階戻す
- message 不整合: 継続監視
- 状態更新不整合: 一段階戻す
- Audit Log 更新不整合: 継続監視
- Settings 保存不整合: 継続監視
- fail-safe 不備: 即停止
- LockGuard 誤作動: 継続監視

#### Level1 参照系 停止/巻き戻し判定（詳細）

- 一覧が表示されない: 即停止
  - 判定目安: 初期表示または再取得後に一覧が空白のまま復帰しない
- 詳細が表示されない: 即停止
  - 判定目安: 一覧選択に対して詳細パネルが更新されない状態が継続する
- Dashboard が空または明らかに不整合: 即停止
  - 判定目安: 一覧件数・状態内訳とDashboard表示が一致しない状態が続く
- Audit Log 一覧が崩れる: 即停止
  - 判定目安: ログ表示の欠落・列崩れ・順序崩れで参照不能になる
- Settings 参照が崩れる: 条件付き停止
  - 判定目安: 単発は継続監視、同一事象が再発したら停止
- message がまったく出ない: 即停止
  - 判定目安: 参照系の成功/失敗結果に対してmessage表示がゼロ件
- プレースホルダー復帰しない: 条件付き停止
  - 判定目安: 単発は継続監視、再発時は停止して一段階戻す判断を行う

#### Level1 条件付き継続 追加監視条件

- 一覧表示
  - 継続条件: 一覧が継続表示され、再読込後も表示欠落がない
  - 追加で見るべき指標: 行欠落件数、空白化回数、再表示所要時間
  - 監視頻度: 監視サイクルごとに3回確認
  - 継続中止条件: 一覧消失または行欠落が連続発生
- 詳細表示
  - 継続条件: 一覧選択と詳細表示の対象一致が維持される
  - 追加で見るべき指標: 選択不整合件数、更新遅延、表示失敗回数
  - 監視頻度: 監視サイクルごとに3回確認
  - 継続中止条件: 詳細未表示または対象不整合が再発
- Dashboard
  - 継続条件: 一覧件数・状態内訳とDashboard表示が一致する
  - 追加で見るべき指標: 数値乖離件数、空欄化回数、更新遅延
  - 監視頻度: 監視サイクルごとに3回確認
  - 継続中止条件: 数値空欄化または不整合継続
- Audit Log 参照
  - 継続条件: ログ一覧の行/列/順序が維持され読取可能
  - 追加で見るべき指標: 行崩れ件数、表示欠落件数、順序不整合件数
  - 監視頻度: 主要確認操作ごとに毎回確認
  - 継続中止条件: 読取不能な崩れ・欠落・順序崩れが再発
- Settings 参照
  - 継続条件: 参照値が空欄化せず安定表示される
  - 追加で見るべき指標: 空欄化回数、初期値固定回数、再取得後一致率
  - 監視頻度: 監視サイクルごとに3回確認
  - 継続中止条件: 空欄化または初期値固定が連続発生

### Level2: 低リスク更新系

- 再描画崩れ: 一段階戻す
- message 不整合: 一段階戻す
- 状態更新不整合: 一段階戻す
- Audit Log 更新不整合: 一段階戻す
- Settings 保存不整合: 継続監視
- fail-safe 不備: 即停止
- LockGuard 誤作動: 継続監視

#### Level2 低リスク更新系 停止/巻き戻し判定（詳細）

- Settings 保存後に値が戻る: 即停止
  - 判定目安: 保存成功後の再読込で設定値が戻る
- Connect しても状態が変わらない: 即停止
  - 判定目安: 実行後も接続状態が更新されず継続する
- Disconnect しても状態が変わらない: 即停止
  - 判定目安: 実行後も切断状態が更新されず継続する
- Audit Log が更新されない: 条件付き停止
  - 判定目安: 単発欠落は継続監視、再発または連続欠落で停止
- message が不整合: 条件付き停止
  - 判定目安: 単発不整合は継続監視、同一不整合の再発で停止
- 再描画が崩れる: 即停止
  - 判定目安: 一覧・詳細・Dashboard・Audit Log の崩れが復帰しない
- 一覧/詳細/管理パネルの状態が食い違う: 即停止
  - 判定目安: 同一対象の状態が3画面で一致しないまま継続する

#### Level2 条件付き継続 追加監視条件

- Settings 保存
  - 継続条件: 保存成功後の再読込で値が保持される
  - 追加で見るべき指標: 値巻き戻り件数、Audit Log 更新時間、 message 一致率
  - 監視頻度: 毎回 Settings 保存後に即座に再読込確認
  - 継続中止条件: 保存後の値巻き戻りが再発
- Connect
  - 継続条件: 実行後に接続状態が詳細/一覧/管理パネルで一致する
  - 追加で見るべき指標: 状態不整合件数、表示反映時間、Audit Log 更新遅延
  - 監視頻度: 毎回 Connect 実行後に3画面で即座に確認
  - 継続中止条件: 接続状態の不整合が継続
- Disconnect
  - 継続条件: 実行後に切断状態が詳細/一覧/管理パネルで一致する
  - 追加で見るべき指標: 状態不整合件数、表示反映時間、Audit Log 更新遅延
  - 監視頻度: 毎回 Disconnect 実行後に3画面で即座に確認
  - 継続中止条件: 切断状態の不整合が継続
- 一覧表示（巻き込み）
  - 継続条件: Level2 操作後も行欠落・表示崩れなし
  - 追加で見るべき指標: 行欠落件数、空白化回数、再描画所要時間
  - 監視頻度: 各操作直後に毎回確認
  - 継続中止条件: Level2 操作後の一覧消失・行欠落が再発
- 詳細表示（巻き込み）
  - 継続条件: Level2 操作後も選択対象と詳細表示が一致
  - 追加で見るべき指標: 対象不整合件数、更新遅延、表示失敗回数

## Level4 発生時の停止条件（2026-03-24）

> Level4 は最終危険操作段階。疑わしい兆候は全て即停止を基本とする。

### confirmed チェック無しで Export/Delete が動く
- 判定ラベル: 即停止
- 判定目安: confirmed 未チェックで API 呼び出し・実行が進んだ時点

### password / twoFactorCode / confirmationText 不足でも動く
- 判定ラベル: 即停止
- 判定目安: 認証フィールドの欠損・未検証が1つでも存在した状態で実行が完了した時点

### LockGuard が閉じない / 誤作動する
- 判定ラベル: 即停止
- 判定目安: 実行完了後も LockGuard が開いたまま、または途中で意図せず閉鎖した時点

### Export / Delete 後に状態が不整合
- 判定ラベル: 即停止
- 判定目安: 実行成功後に一覧・詳細・管理パネルの状態が一致しない場合

### Audit Log が更新されない
- 判定ラベル: 即停止
- 判定目安: Export/Delete の実行後（成功・失敗問わず）ログ追記が1件も確認できない場合

### message が不整合
- 判定ラベル: 条件付き停止
- 判定目安: 成功/失敗の逆表示は即停止。message 無表示は1回発生で停止

### 再描画が崩れる
- 判定ラベル: 条件付き停止
- 判定目安: 画面消滅・空白化は即停止。軽微な表示ズレは1回発生で停止

### 一覧/詳細/管理パネルの状態が食い違う
- 判定ラベル: 即停止
- 判定目安: 同一対象の状態が3画面で一致しない時点

### Level1 / Level2 / Level3 解放機能の巻き込み崩れ
- 判定ラベル: 即停止
- 判定目安: Export/Delete 後に既存解放機能（一覧/詳細/Dashboard/Audit Log/Settings/Connect/Disconnect/Freeze/Unfreeze）のいずれかに崩れが確認された時点

## Level4 LockGuard・認証導線 安定監視ポイント（固定）

> Freeze / Unfreeze の警告表示導線とは混同せず、Export / Delete 専用導線として確認する。

- LockGuard が正しく開くか
  - 正常: Export / Delete 開始時に対象操作用 LockGuard が1回だけ表示される
  - 異常兆候: 開かない、別操作向け文言が出る、二重表示される
- キャンセルで正しく閉じるか
  - 正常: キャンセルで LockGuard が閉じ、状態変更が発生しない
  - 異常兆候: 閉じない、閉じた後も状態が変化する、背景操作が不能になる
- confirmed チェック未実施時に実行されないか
  - 正常: confirmed 未チェックでは実行要求が通らない
  - 異常兆候: 未チェックで API 呼び出しまたは実行完了が発生する
- password 不足で止まるか
  - 正常: password 未入力時は実行中断し、状態変更が起きない
  - 異常兆候: password 未入力でも実行される
- twoFactorCode 不足で止まるか
  - 正常: twoFactorCode 未入力時は実行中断し、状態変更が起きない
  - 異常兆候: twoFactorCode 未入力でも実行される
- confirmationText 不一致で止まるか
  - 正常: confirmationText 不一致時は実行中断し、状態変更が起きない
  - 異常兆候: 不一致でも実行される、誤った対象に適用される
- 実行後に LockGuard が残らないか
  - 正常: 成功/失敗いずれでも処理後に LockGuard が閉じる
  - 異常兆候: 処理後も残留する、再実行不可になる
- 他画面に誤影響が出ないか
  - 正常: 一覧・詳細・Dashboard・Audit Log・Settings・Connect/Disconnect・Freeze/Unfreeze が崩れない
  - 異常兆候: 表示崩れ、状態食い違い、操作不能が発生する

## 定期レビュー運用フェーズ 停止・巻き戻し判断条件（固定）

| 事象 | 判定目安 |
|---|---|
| 一覧/詳細表示崩れ（操作系起因、複数箇所・再発） | **全面見直し** |
| 一覧/詳細表示崩れ（単発・参照系のみ・自己回復） | 継続監視 |
| Dashboard 不整合（操作後の件数・状態サマリが反映されない・複数回） | 機能単位で巻き戻し |
| Dashboard 不整合（一時的な遅延・自己回復） | 継続監視 |
| Audit Log 更新異常（action/target/result 欠落・1件以上） | 機能単位で巻き戻し |
| Audit Log 更新異常（危険操作の記録欠落） | **全面見直し** |
| Settings 保存不整合（保存後に値が元に戻る・再発） | 機能単位で巻き戻し |
| Settings 保存不整合（単発・即時自己回復） | 継続監視 |
| Connect / Disconnect 不整合（status が3画面で食い違う・残留あり） | 機能単位で巻き戻し |
| Freeze / Unfreeze 不整合（3画面 status 不一致・Audit Log 欠落） | 機能単位で巻き戻し |
| Freeze / Unfreeze 不整合（confirmed すり抜け） | **全面見直し** |
| Export / Delete 不整合（実行結果と message / Audit Log / 3画面が食い違う） | **全面見直し** |
| Export / Delete 不整合（Audit Log 欠落のみ） | 機能単位で巻き戻し |
| LockGuard 不備（閉じない・残留・処理後も開いたまま） | **全面見直し** |
| 認証不備（認証不足で危険操作が通過） | **全面見直し** |
| fail-safe 未発動（危険操作が保護なく進む） | **全面見直し** |
| message 不整合（操作結果と逆・前回残留・複数回） | 機能単位で巻き戻し |
| message 不整合（単発・軽微・自己回復） | 継続監視 |
| 再描画崩れ（操作後に画面が空白化・情報欠落・複数回） | 機能単位で巻き戻し |
| 再描画崩れ（一時的な遅延のみ・自己回復） | 継続監視 |
| 原因不明の多箇所不整合（複数機能に同時発生） | **全面見直し** |

### 定期レビュー停止・巻き戻し後の対応方針

- **全面見直し**: 即時すべての条件付き運用機能を停止・巻き戻し、根本原因が特定されるまで再開しない
- **機能単位で巻き戻し**: 該当機能のみ条件付き運用または保留に戻し、他機能は継続監視のまま次回レビューを待つ
- **継続監視**: 単発・自己回復事象は記録のみ行い、次回定期レビューで再発有無を確認する
