## 7. 実施の流れ

### ステップ 1: GitHub 管理画面確認（5分）
1. GitHub リポジトリ → Settings → Actions → General で permissions を確認
2. .github/workflows/python-check.yml の permissions セクションを確認
3. 結果を 1-1〜1-4 に記入

### ステップ 2: WordPress 管理画面確認（5分）
1. WordPress 管理画面 → ユーザー で WP_USERNAME に対応するユーザーを開く
2. ロール（権限）を確認
3. Application Passwords の設定詳細を確認
4. 結果を 2-1〜2-4 に記入

### ステップ 3: Slack 設定確認（5分）
1. Slack 管理画面 → Apps & integrations → Incoming Webhooks で SLACK_WEBHOOK_URL を開く
2. 接続先チャンネルと用途を確認
3. チャンネルが用途専用かどうか確認
4. 結果を 3-1〜3-4 に記入

### ステップ 4: 結果をサマリに集約（5分）
1. セクション 5 にまとめる
2. セクション 6 で判定を記入
3. preflight_checklist.md への反映を判断

**合計所要時間: 約 20分**

## 注記
- 値そのもの（URL、パスワード）は記載しない
- 名前、用途、権限レベルのみ記録
- 不明な場合は [ ] のまま残し、後日専門家に相談
- fail-safe 優先：過剰権限の疑いがあれば、その場で Go 判定には進めない
